## papyr-pdf-layout-editor Cloudflare Workers deploy 対応

今日は `target-repos/papyr-pdf-layout-editor.git/.wt/codex/cloudflare-workers-deploy` だけを変更対象にする。`.wt/main` は baseline なので触らない。

実装計画:

- `wrangler.toml` と一時 deploy script を廃止し、`wrangler.jsonc` を Cloudflare Workers deploy 設定の正にする。preview は `papyr-pdf-layout-editor-preview`、production は `papyr-pdf-layout-editor`、workers.dev、Assets binding、SPA fallback、observability を設定する。
- `wrangler types` の生成結果を使って `src/worker.ts` の `Env` 手書きをやめ、Worker binding の型を config 由来に寄せる。
- `package.json` と `justfile` に worker type 生成、dry-run、preview / production deploy の script を追加し、Cloudflare 認証が必要な deploy 系は `opz` 経由で実行できるようにする。
- GitHub Actions に validation と手動 deploy workflow を追加し、Cloudflare の credential は raw GitHub Secrets ではなく 1Password / `opz` runtime injection で渡す前提を README に書く。
- `.gitignore` に local env ファイルを追加し、secret 値を表示・埋め込みしない。

検証計画:

- `pnpm install`
- `pnpm run typecheck`
- `pnpm run test`
- `pnpm run build`
- `pnpm run worker:types:check`
- Cloudflare 認証なしで可能な範囲の `wrangler deploy --dry-run`。認証が必要で止まる場合は blocker を記録する。

実施結果:

- `wrangler.jsonc` に preview / production env、workers.dev、Assets binding、SPA fallback、observability を集約した。
- 一時的に TOML を生成する deploy script と旧 TOML 設定を削除し、deploy は `wrangler deploy --env ...` と `opz run --environment ...` の組み合わせに寄せた。
- `wrangler types` で `src/worker-configuration.d.ts` を生成し、Worker は生成された `Env` を使う形にした。これは config から binding 型を作る仕組みで、手書き型の drift を避けるため。
- GitHub Actions は通常 validation と手動 deploy を分け、Cloudflare の API token / account id は GitHub Secrets ではなく 1Password Environment から実行時注入する前提にした。
- 検証は typecheck / test / build / Worker types check / preview と production の dry-run / e2e まで通した。dry-run は認証なしでも upload plan と binding 確認まで進んだ。

## moonrepo ral related repo teams

moonrepo 自体に、関連の強い target repo 群を `ral` team として扱うための harness を追加する。`ral` は task state の正ではなく、短い追従連絡と changelog 共有の補助に限定する。task state は引き続き `.codex/tasks/*.json`、git、GitHub を正にする。

変更対象は `repository.ini` / `repository.ini.example`、`scripts/repo-lib.sh`、新規 `scripts/rally-groups.sh`、`justfile`、`README.md`、`AGENTS.md`、この diary にする。

実装計画:

- `repository.ini` を section 対応にし、active repo は `[repositories]`、ral group は `[rally.teams]` に書けるようにする。既存の section なし `<owner>/<repo>` 行は後方互換として active repo 扱いにする。
- 初期 group は `papyr = papyr.mbt, papyr-docs.mbt, blog.mbt` と `mhx = mhx.mbt, mhx_spec.mbt` にする。`htmx.mbt` は現時点では active / clone に無いので含めない。
- `scripts/rally-groups.sh` を追加し、group の list / show / validate / join / inbox / send / changelog を扱う。team 名は `repo-group-<group>`、agent 名は repo basename、親は `moonrepo` にする。
- `justfile` に `rally-groups`、`rally-group`、`rally-group-join`、`rally-group-inbox`、`rally-group-send`、`rally-changelog` を追加する。
- changelog は `docs/rally/<group>/<YYYY-MM-DD>-<slug>.md` に markdown として残し、`ral` が使える場合だけ同じ summary を team に送る。未導入時は `just rally-install` の hint を出す。

検証計画:

- `bash -n scripts/repo-lib.sh scripts/repo-targets.sh scripts/rally-groups.sh`
- `just --list`
- `just doctor`
- `just rally-groups`
- `just rally-group papyr`
- `just rally-group mhx`
