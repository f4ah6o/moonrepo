## 学び・メモ

- 対象 repo の定義が `repository.ini` と `repos/` 直走査で分かれていると、clone の残骸だけで一括運用の結果が変わってしまう。
- `moon-dst` は `--ignore` を複数渡せるので、active 対象だけを守りたい時は root 全体を走らせつつ inactive clone を除外できる。
- destructive な recipe は、名前を変えなくても `FORCE=1` の明示だけでかなり事故を減らせる。
