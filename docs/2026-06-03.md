# 2026-06-03

## FWD.mbt dslx.mbt L1 DSL 再構築

今日は `FWD.mbt` の dslx 連携を、契約チェックだけの段階から L1 DSL の実コンパイル経路へ進める。L1 DSL は最初の実用 slice という意味で、FWD の全機能ではなく `schema`、`state`、`entity`、`transition`、`effect`、`rule` を使って IR 生成まで通す最小の入力言語として扱う。

実装は target repo の `.wt/main` を直接変更せず、専用 worktree で進める。まず `dslx.mbt` 側で `@dslx.fwds()` grammar を FWD L1 DSL 用に拡張し、examples と parser contract のテストを追加する。次に `FWD.mbt` 側で `compile_dslx_l1(input : String) -> Result[@ir.FwdIR, CompileError]` を追加し、dslx の parse tree から `@schema.Schema` を組み立てて、既存の `resolve_schema`、`validate_schema`、`emit_ir_from_schema` を通す。

変更対象は主に `dslx.mbt` の grammar/example/test と、`FWD.mbt` の `src/compiler` package である。既存 YAML compiler、CLI、runtime、versioned API は互換性維持のため変更しない。検証は各 repo で `moon check` と `moon test` を行い、FWD では同等 YAML sample との IR JSON 比較、missing state/transition/effect、unknown type、duplicate rule、builtin rule conflict の失敗系も追加する。

## n8n.mbt dslx contract follow-up merge

`n8n.mbt` の PR #2 は、dslx contract 適用後に MoonBit の `Show` 表示が `String("x")` から `String(x)`、`NodeId("x")` から `NodeId(x)` へ変わったことへ inspect 期待値を追従する差分である。merge 前に `moon check`、`moon test src/dslx_contract_wbtest.mbt`、`moon test`、`git diff --check origin/main...HEAD` を通し、opencode-review-loop で exact `LGTM` を確認した。

PR は draft を解除して squash merge し、`target-repos/n8n.mbt.git/.wt/main` を `origin/main` に fast-forward した。fetch の途中で disk 空き不足が出たため、再生成可能な npm cache の `~/.npm/_cacache` を削除して空きを確保した。task manifest は `merged` に更新し、専用 worktree は clean なまま残している。
