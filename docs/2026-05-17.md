## 実施メモ

`just clone` は bare repo を `target-repos/<repo>.git` に作り、既定ブランチの作業 tree を `.wt/main` に作る形にした。bare repo には `wt.basedir=.wt` を設定する。`git-wt` は必須コマンドとして `doctor` で確認するが、recipe 内の worktree 作成は既存 script との整合性を優先して `git worktree add` を使う。

`codex-start` は root の `worktrees/` ではなく、対象 bare repo 内の `.wt/codex/<task-slug>` に作業 tree を作る。manifest の `worktree` path は新しい場所を指すので、`codex-status` と `codex-pr` はそのまま追従できる。

`moon-dst` の一括 recipe は `target-repos/` 全体を root scan しない。bare repo の内部や他 task worktree を巻き込むのを避けるため、対象 worktree ごとに `moon-dst scan/apply/just --root <path>` を実行する。

## OpenUI.mbt 初期実装の計画

`f4ah6o/openui.mbt` を新規 target repo として作り、MoonBit で OpenUI Lang v0.5 の state までを扱う runtime を実装する。最初の renderer は Rabbita 向けに絞る。

変更する主なファイル:

- `repository.ini`: `f4ah6o/openui.mbt` を active repo に追加する。
- `target-repos/openui.mbt.git/.wt/codex/openui-v05-rabbita-runtime`: MoonBit module、parser、validator、state runtime、Rabbita renderer、README、CI を追加する。
- `scripts/codex-task.sh`: `target-repos/<repo>.git/.wt/codex/<task>` の path 組み立てで bare repo path を先に解決する。

`OpenUI Lang` は UI を component call の列として表す小さな DSL で、ここでは parse した木を Rabbita の HTML 表現に変換する。`state` は `$name = value` の形で宣言される値で、`@Set` と `@Reset` から更新できる最小 runtime として持つ。
