# 2026-06-02

## dslx.mbt 新規作成計画

今日は `f4ah6o/dslx.mbt` を新規 target repo として作り、MoonBit で DSL 生成用メタフレームワークの動く MVP を実装する。`target-repos/dslx.mbt.git/.wt/main` は baseline checkout として固定し、初期 repo 作成後の実装は新規 repo の初期 commit としてまとめる。

変更対象は moonrepo 側では `repository.ini` とこの日誌だけにする。既存の `docs/2026-05-30.md` と `docs/2026-05-31.md` には未コミット変更があるため、今回の作業では触らない。

`dslx.mbt` 側は Apache-2.0 の MoonBit module として初期化し、`dslx/core`、`dslx/astx`、`dslx/pars`、`dslx/vald`、`dslx/prnt`、`dslx/emit`、`dslx/test` の package を作る。公開 API は 4 文字名に統一し、`dslx`、`node`、`case`、`fldx`、`rule`、`seqx`、`altx`、`many`、`optx`、`litx`、`tokn`、`rgxx`、`prat`、`pref`、`infx`、`post`、`vald`、`chec`、`warn`、`fail`、`hint`、`diag`、`span`、`prnt`、`grup`、`line`、`join`、`nest`、`brkx`、`emit`、`file`、`mods` などを builder として提供する。

MoonBit の `test` は構文語で、この toolchain ではバッククォートによる公開関数名にもできなかったため、Test DSL の入口は 4 文字の `tstx` にする。同様に top-level `init` も特別扱いされるため、state machine の初期状態指定は method `.init(...)` として提供する。

MVP では実ファイルを書き込む codegen は作らない。`emit(...).file(...).mods(...)` は仮想ファイルの path と content を返すだけにし、生成結果をテストしやすくする。検証は `moon check`、`moon test`、`moon info` を実行し、公開 API の `pkg.generated.mbti` を確認する。

## dslx.mbt v0.2 完成度向上計画

次の作業では `dslx.mbt` の MVP を設計整理重視で v0.2 相当に引き上げる。`target-repos/dslx.mbt.git/.wt/main` は baseline のまま固定し、`just codex-start dslx.mbt completion-v02` で専用 worktree を作る。

変更対象は target repo 側の root package 実装、README、テスト、`pkg.generated.mbti` にする。既存の `dslx/core`、`dslx/astx`、`dslx/pars`、`dslx/vald`、`dslx/prnt`、`dslx/emit`、`dslx/test` は placeholder なので、root API と重複するだけなら削除し、README の推奨構成も実態に合わせる。

公開 API は型、関数、field、method まで 4 文字名を守る。`test` と top-level `init` は MoonBit の構文上の制約があるため、Test DSL は `tstx`、state machine の初期状態は method `.init(...)` を維持する。

実装内容は `litx`、`tokn`、`rgxx`、`seqx`、`altx`、`many`、`optx` を文字列入力に対して実行する簡易 parser、spec/rule/node の基本 validation、安定した pretty printer、virtual file codegen の強化にする。完全な regex engine と Pratt parser 実行器は今回の範囲外にする。

検証は `moon fmt`、`moon check`、`moon test`、`moon info` を実行し、`pkg.generated.mbti` に 4 文字超の公開 API が残っていないことを確認する。
