- `#13` の first pass として publication projection を追加した。ここでいう projection は、decoded document 群から docs site が読む `manifest.json` 相当を組み立てる処理を指す。
- projection の詰まりどころは sort order だった。upstream TypeScript は `localeCompare()` を使っていて、MoonBit の単純な文字列比較では `BackendAdapter` と `backend につなぐ` のような順序がずれた。
- 対応として `src/core/moon.pkg` の target 分岐を使い、JS backend だけ `localeCompare()` を使う `collation_js.mbt`、それ以外は lexical fallback を使う `collation_fallback.mbt` に分けた。これで frozen `manifest.json` と一致しつつ `moon check` も維持できた。
- 2026-05-02 時点の確認は `moon check` と `moon test --target js`。manifest fixture test まで green。
- `#14` の first pass として docs-build CLI を追加した。ここでの artifact bundle は、`documents.json`、`manifest.json`、`search-index.json`、`r2/**`、`public-data/**` を file path と content の配列として返す machine-readable な出力を指す。
- docs-build は current docs style の `documents.json` envelope と raw markdown directory、caller-supplied `generatedAt` を受ける。`generatedAt` を caller から渡すのは、golden test を wall-clock time から切り離して byte-stable に保つため。
- JSON artifact は JS target で `JSON.stringify(value, null, 2)` 相当の pretty render を使い、non-JS target では buildability を優先して fallback renderer を使うようにした。これで fixture parity と cross-target `moon check` を両立した。
